# 赛博世界愿景

## 愿景

我希望做一个**通用的**赛博世界框架，它可以被实例化为任意的组织形态，如：

- 🏛️ **封建王朝** —— 皇帝、丞相、太尉，等级森严的治理体系
- 🏢 **科技公司** —— CEO、CTO、产品经理，现代企业管理结构  
- 🎰 **地下赌场** —— 庄家、荷官、保安，灰色产业的运作机制
- 🏫 **魔法学院** —— 院长、教授、学生，奇幻世界的师徒传承
- 🏴‍☠️ **海盗船队** —— 船长、大副、水手，冒险者的松散联盟
- ...以及任何你能想象的组织形态

**核心思想**：框架提供通用的 Multi-Agent 协作基础设施，具体的角色、规则、流程由配置文件和 Prompt 定义，无需改动代码即可创造全新的赛博世界。

---

## 原则

做事分三个阶段：

```
🔍 调研 → 📐 设计 → 🔨 实现
```

**每次只聚焦一个阶段！**

- **调研阶段**：充分了解现有方案、分析问题、收集需求
- **设计阶段**：完成架构设计、接口定义、数据结构，不碰代码
- **实现阶段**：按设计文档编码、测试、部署

禁止跨阶段提前行动，确保每个阶段都有完整的思考和沉淀。

## 当前任务

参考 ai-toolbox 的实践经验，完成赛博世界框架的**初步设计**，**不需要去实现**！

### ai-toolbox 经验要点

1. **三层架构**：SYSTEM_PROMPT(导航) + Rules(约束) + Skills(流程)
2. **动态组合**：根据场景按需加载，精确控制上下文
3. **记忆机制**：Session/Channel/Bot/Org/User 五级分层
4. **工具使用**：代码执行、网页搜索等能力可插拔
5. **配置驱动**：YAML 定义角色，无需代码即可扩展
