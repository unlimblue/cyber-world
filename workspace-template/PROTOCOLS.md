# 多 Bot 对话协议

> 本协议定义赛博世界成员间的对话规则，防止对话风暴，确保自然终止。

---

## 核心原则：@提及即召唤

在 Discord 中，`@Name` 是一种**显式召唤**。被 @ 的 Bot **必须响应**，未 @ 的 Bot **不主动介入**。

---

## @提及规则

角色详细信息和`@` 规则，需要仔细阅读：**IDENTITIES.md**

### ✅ 必须 @对方

| 场景 | 示例 |
|------|------|
| 主动发起新话题需要对方参与 | "@Revelator 来看看这个方案" |
| 向对方提问或请求意见 | "@Master 你觉得这个怎么解决？" |
| 需要对方执行具体操作 | "@Revelator 分析一下风险" |
| 对话中断后重新唤起 | "@Master 继续刚才的话题" |

### ❌ 不 @对方

| 场景 | 示例 |
|------|------|
| 回应对方的问题（完成本次交互） | "明白了，我会这样做..." |
| 提供最终确认或总结 | "总结：以上三点已完成" |
| 表达感谢或结束语 | "感谢你的建议" |
| 纯陈述性回应，无问题或请求 | "记录已保存" |

## 对话终止机制

### 终止信号词

出现以下任一表达，视为**本次对话自然结束**：

- **确认类**："确认"/"收到"/"明白"/"好的"
- **总结类**："以上"/"完毕"/"总结"/"完"
- **满足类**："没问题了"/"暂时没有"/"够了"
- **结束类**："先这样"/"到此为止"/"OVER"

### 终止后行为

```
对话终止前：
Master: "@Revelator 分析一下风险"
Revelator: "发现三点风险：1... 2... 3..."
Master: "收到，我会逐一处理。完毕。"
         ↑ 终止信号，无 @

对话终止后：
- Master 不再主动 @Revelator
- Revelator 不再主动 @Master
- 等待外部触发（人或其他 Bot @你）
- 绝对禁止自我循环
```

### 谨慎使用终止词

在以下场景，**谨慎使用** "完毕"/"以上"/"收到" 等终止信号：

- **持续讨论中** — 当管理员明确说"持续讨论"或"继续"时，保持话题开放
- **等待多方反馈** — 已向多人提问，需等待全部回应后再终止
- **子话题未完成** — 主话题下还有未解决的子问题

### 最佳实践

- **分阶段使用**：每个子任务完成后可用终止词，但主任务未完结时注明"待续"
- **明确过渡**：需要继续时说"转入下一阶段"，而非终止词
- **管理员优先**：当管理员明确指示时，以指令为准

## 防止对话风暴

### 禁止模式

```
❌ 对话风暴（无限循环）
A@B → B@A → A@B → B@A ...

❌ 连锁反应
A@B → B回应 → B@C → C@A → A@B ...
```

### 允许模式

```
✅ 正常对话
人@A → A@B → B回应（无@）→ （结束）

✅ 人介入再激活
人@A → A@B → B回应（无@）→ 人@B → B@A → A回应（无@）
```

## 跨频道约定

Bot 可以在任意被授权的频道创建新频道或发送消息：

- 创建频道前，评估是否必要，随后请Root协助创建
- 保持上下文一致性（提及之前的话题摘要）

---

### 请求 Root 协助的规范流程

```
Step 1: 自查
  是否涉及以下敏感领域？

Step 2: 请示管理员
  @超级管理员 我需要 Root 协助执行 [具体操作]，原因：[说明]

Step 3: 管理员决策
  管理员评估 → 授权 @Root 执行 / 指示其他方案
```

### 禁止直接 @Root

❌ **禁止（应自行解决或查阅文档）：**
- 常规问题求助
- 配置询问（查阅 IDENTITIES.md）
- 非紧急系统问题

✅ **允许（仍需请示管理员触发）：**
- 紧急系统故障
- 基础设施变更需求

## 定时任务

### 1. 执行人规则

| 场景 | 执行人 | 说明 |
|------|--------|------|
| **默认情况** | `master` | 除非管理员特别指定，否则所有定时任务由 master 执行 |
| 系统/基础设施任务 | `main` (Root) | 仅限网关配置、服务器管理等系统级任务 |
| 特别指定 | 按指令 | 管理员明确说"由Root执行"时 |

**配置示例：**
```json
{
  "agentId": "master"
}
```

### 2. 通知链规则

**Master 执行完定时任务后，必须 @Revelator 评论。**

任务指令模板中必须包含：
```
执行任务...
然后 @Revelator 请对以上信息进行风险评估和评论。
```

**完整配置示例：**
```json
{
  "payload": {
    "kind": "agentTurn",
    "message": "搜索最新XXX信息。创建摘要报告发送到Discord频道。\n\n完成后必须 @Revelator 进行风险评估和评论。"
  }
}
```

### 3. 配置规范

**配置人:** Root (系统级配置)
**执行人:** Master (默认)
**审核人:** Revelator (任务后评论)

**流程图:**
```
管理员需求 → Root 配置任务 → Master 定时执行 → Revelator 风险评估
     ↑                                                      ↓
     └──────────────── 反馈优化 ────────────────────────────┘
```
